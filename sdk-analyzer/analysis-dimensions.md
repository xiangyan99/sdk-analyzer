# Analysis Dimensions for Azure Python SDK Packages

## SDK Package Structure

```
{package-root}/{namespace}/
  __init__.py
  _patch.py              # Top-level customizations
  _*.py                  # Hand-written utility modules
  _generated/            # AUTO-GENERATED — reference only
    __init__.py
    _client.py
    _configuration.py
    models/
    _operations/
    aio/
  models/
    __init__.py
    _patch.py            # Model customizations
  operations/
    __init__.py
    _patch.py            # Operations customizations
  aio/
    __init__.py
    _patch.py            # Async customizations
    operations/
      __init__.py
      _patch.py          # Async operations customizations
```

## How to Identify Customization Files

- **Generated:** inside `_generated/`, or (if no `_generated/` exists) files containing `"Code generated by Microsoft (R) Python Code Generator"`.
- **Customization:** everything else outside `_generated/` that is not `__init__.py`, `_version.py`, or an empty stub.
- A `_patch.py` is **empty** (skip it) if it only has the copyright header, the default docstring, `__all__: List[str] = []` with no entries, and an empty `patch_sdk()`. If `__all__` has entries or the file defines classes/functions beyond `patch_sdk()`, it is **non-empty**.

If a scan target does not exist, record it as absent and move on.

## Dimension 1: Customization Inventory

Covers clients, models, operations, and utilities in a single pass.

**Scan targets:** All non-empty `_patch.py` files and all `.py` files outside `_generated/` (excluding `__init__.py`, `_version.py`).

**What to extract for each file:**

1. Classes — name, base class, purpose (client, model, policy, helper, etc.)
2. Key methods — overrides of generated methods, new public methods, with signatures
3. Constructor changes — added/modified parameters, credential wrapping, policy injection
4. LRO/paging customizations — custom polling or pagination behavior
5. Constants and enums — module-level constants, hand-maintained `ApiVersion` enum if present

**Record format:**

```
File: {relative path}
  Category: {client | model | operations | utility}
  Classes: {name (base) — purpose}
  Key methods: {name — override|new, one-line summary}
  Constructor changes: {description or "none"}
  LRO/Paging: {description or "none"}
  Constants: {name = value}
```

## Dimension 2: Public API & Async Parity

**Scan targets:** `__init__.py` at each level; sync/async file pairs.

**What to extract:**

1. **Exports** — symbols re-exported from `_patch.py`, symbols added beyond `_generated/`, symbols removed vs `_generated/__init__.py`
2. **Async parity** — for each sync customization file, check the async counterpart (`_patch.py` ↔ `aio/_patch.py`, `operations/_patch.py` ↔ `aio/operations/_patch.py`). Status: `equivalent`, `divergent`, or `missing`.

**Record format:**

```
Exports:
  File: {__init__.py path}
    Patch re-exports: [symbols]
    Added: [symbols]
    Removed: [symbols]

Async parity:
  {sync_file} <-> {async_file}: {equivalent | divergent | missing} — {differences or "none"}
```

## Dimension 3: Import Dependencies & Risks

Identifies fragile links between customization files and generated code. Drives the verification steps in the generated skill.

**Scan targets:** All non-empty `_patch.py` files and all utility files.

**What to extract:**

1. **Import map** — for each customization file, every symbol imported from generated modules. Record exact import statements.
2. **ApiVersion enum** — location, members, default value (or "not found").
3. **Monkey-patches** — runtime modifications to generated objects (e.g., `GeneratedEnum.OldName = GeneratedEnum.NEW_NAME`).
4. **Named patterns** — each distinct customization technique (e.g., "Enum Aliases", "Custom Pagination", "Field Builders"). Record: name, one-sentence description, files, generated symbol dependencies.

**Record format:**

```
File: {relative path}
  Generated imports: [{module}: {symbol1, symbol2, ...}]

ApiVersion:
  Location: {file path or "not found"}
  Members: [{version_name} = "{version_string}", ...]
  Default: {DEFAULT_VERSION value or "N/A"}

Monkey-patches:
  {target_class}.{attribute} = {value} (in {file})

Named patterns:
  {PatternName}: {description}
    Files: [{file1, file2}]
    Depends on: [{generated_symbol1, generated_symbol2}]
```
